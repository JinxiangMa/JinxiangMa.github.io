  • GCC

    Linux编译工具集

    多种语言，交叉编译

    GCC工具集不仅能编译C/C++语言，其他例如Object-C、Pascal、Fortran、Java、Ada等均能进行编译

  • 编译程序的基本知识:GCC编译器对程序的编译分为四个阶段：预处理、编译和优化、汇编、链接。
  • 单个文件编译成执行文件：gcc 文件名

    自动编译的过程包括头文件扩展、目标文件编译、以及链接默认的系统库生成可执行文件，最后生成系统默认的可执行程序a.out。
    中间文件作为临时文件存在，在可执行文件生成后，会删除中间文件。

  • 编译生成目标文件：目标文件是指经过编译器的编译生成的CPU可识别的二进制代码，因为其中的一些函数过程没有相关的指示和说明，目标文件不能执行。 默认情况下生成的目标文件的文件名和源文件的名称一样，只是扩展名为.o。
  • 多文件编译:
  • 预处理:将源文件中的头文件包含进源文件中，并且将文件中定义的宏进行扩展。 如果需要指定源文件预编译后生成的中间结果文件名，需要使用选项-o
  • 编译成汇编语言 默认情况下生成的文件名和源文件一致，扩展名为.s

  • 生成动态链接库

    名称：

    别名由一个前缀“lib”，然后是库的名字，再加上一个后缀“.so”构成。
    真名是动态链接库的真实名称，一般总是在别名的基础上加上一个小版本号、发布版本等构成。
    库中函数和变量的地址是相对地址，在动态链接库与程序加载时转为绝对地址

    在动态链接库安装的时候，总是复制库文件到某个目录下，然后用一个软连接生成别名，在库文件进行更新的时候，仅仅更新软链接

    一般将动态链接库复制到系统默认的动态链接库路径下：/etc/ld.so.conf

  • 动态加载库:动态加载库可以用程序的方法来控制什么时候加载库

    动态加载库主要有函数dlopen()(打开动态库)、dlerror()、dlsym()（获得函数指针）和dlclose()来控制动态库的使用
